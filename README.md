## 1. Objective

`MazCluster.exe` is a high-performance clustering application designed for large-scale 2D point cloud data. The core objective is to achieve linear (O(N)) or near-linear (O(N,K)) time complexity, making it suitable for datasets that are too large for traditional algorithms that rely on a full N×N distance matrix.

### Key Features:

*   **Sparse Distance Matrix**: Instead of a memory-intensive N×N matrix, the algorithm uses a sparse distance matrix, storing only the K-nearest neighbors for each node. This dramatically reduces memory consumption.
*   **Multi-Pass Refinement**: The clustering process involves multiple passes to ensure high-quality results:
    *   **Pass 1 (DM Build)**: An optional pass to construct the sparse distance matrix.
    *   **Pass 2 (Seeding & Assignment)**: Initial cluster heads are seeded, and all other nodes are assigned to the nearest head.
    *   **Pass 3 (Refinement)**: The algorithm refines the clusters by re-calculating cluster centers and re-assigning nodes to their new nearest cluster head.
*   **Parallel Processing**: The C# implementation is parallelized using the .NET `ThreadPool` to leverage multi-core processors for significant speedups.
*   **Headless & GUI Modes**: The executable can be run in a headless command-line mode for scripting and experiments, or as a standalone GUI application for interactive visualization.

## 2. Project File Descriptions

This project is composed of a Python GUI and a C# clustering engine.

### Python GUI & Scripts

*   `MegaClusterHero.py`: The main entry point for the graphical user interface (GUI). This `tkinter` application provides a dashboard to run experiments and visualize results.
*   `data_gen.py`: A script that runs the "N-Effect" experiments. It calls `MazCluster.exe` with varying numbers of nodes (N) to measure how dataset size affects runtime.
*   `data_gen_k.py`: A script that runs the "K-Effect" experiments. It calls `MazCluster.exe` with varying neighborhood sizes (K) to measure its effect on runtime.
*   `plot_results.py`: Generates the main summary plot (`figure1_runtime_scaling.png`) from the `performance_log.txt` file.
*   `plot_n_effect.py`: Generates a specific plot for the N-Effect analysis (`figure_n_effect.png`).
*   `plot_k_effect.py`: Generates a specific plot for the K-Effect analysis (`figure2_k_effect.png`).

### C# Clustering Engine

*   `MazCluster.exe`: The compiled C# executable that contains the core clustering logic.
*   `MegaClusterHsm/`: The directory containing the C# source code for the `MazCluster.exe` application.
    *   `Program.cs`: The main entry point for the C# application. It handles command-line argument parsing and switches between GUI and headless modes.
    *   `Cluster.cs`: Contains the core implementation of the clustering algorithm, including node generation, the multi-pass clustering logic, and visualization code for the GUI.
    *   `Node.cs`: Defines the `Node` data structure.
    *   `SparseDistanceMatrix.cs`: Implements the memory-efficient sparse distance matrix.

### Data & Results

*   `results/`: This directory is the default location for all output files.
    *   `performance_log.txt`: The main log file where runtime and performance metrics for N-Effect experiments are stored.
    *   `k_effect_log.txt`: The log file for K-Effect experiment results.
    *   `*.png`: Various plots generated by the Python scripts.
    *   `*.csv`: Raw cluster data output by `MazCluster.exe`.

## 3. Quick Start Guide (`MegaClusterHero.py`)

The `MegaClusterHero.py` application is the easiest way to run experiments and see results.

### Prerequisites

*   Python 3.x
*   Required Python libraries: `pandas`, `matplotlib`, `Pillow`
*   .NET Framework (for running `MazCluster.exe`)

### Running the Application

1.  **Open a terminal** in the project's root directory.
2.  **Run the following command**:
    ```bash
    python MegaClusterHero.py
    ```
3.  The application window will open, presenting you with several tabs:
    *   **Full Workflow**: Run the entire data generation and plotting process.
    *   **N-Effect Analysis**: Run experiments focused on the effect of dataset size (N).
    *   **K-Effect Analysis**: Run experiments focused on the effect of neighborhood size (K).
    *   **Results Dashboard**: View summary tables of all your experiment results.
    *   **MazCluster Runner**: A simple interface to run the C# executable directly.

## 4. How to Use `MazCluster.exe`

You can also run the clustering engine directly from the command line. This is useful for scripting or custom experiments.

### Command-Line Usage

```bash
MazCluster.exe [arguments]
```

### Arguments

*   `--nodes <int>`: The total number of data points to generate. (Default: `1000`)
*   `--clusters <int>`: The number of clusters to find. (Default: `10`)
*   `--k <int>`: The neighborhood size to use for the sparse distance matrix.
*   `--output <path>`: The file path to save the output `.csv` file containing node and cluster information.
*   `--logfile <path>`: The file path to save the performance log entry.
*   `--init_mode <string>`: The initialization mode for cluster heads. (Default: `random`)
*   `--energy_log <path>`: The file path to save an energy log (optional).
